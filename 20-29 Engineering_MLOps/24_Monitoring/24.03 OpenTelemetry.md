# OpenTelemetry

## Overview

**OpenTelemetry (OTel)** is a CNCF vendor-neutral open standard for collecting, processing, and exporting telemetry data — **traces, metrics, and logs** — from applications and infrastructure.

> [!INFO] Why OTel?
> OTel decouples instrumentation from backends. Write once, send to any backend (Prometheus, Jaeger, Grafana, Datadog, etc.) via the **OpenTelemetry Protocol (OTLP)**.

## Key Concepts

### Three Pillars
| Signal | What It Tracks | Use Case |
|---|---|---|
| **Traces** | Request flows across services | Latency, distributed debugging |
| **Metrics** | Numeric time-series | Throughput, error rates, resource usage |
| **Logs** | Structured/unstructured events | Error context, audit trails |

### Core Components
- **SDK** — Language-specific instrumentation library (Python, Go, Java, JS, etc.)
- **API** — Stable interface for adding telemetry to code
- **Collector** — Standalone agent/gateway to receive, process, and export signals
- **OTLP** — The wire protocol (gRPC or HTTP/protobuf) for sending telemetry

### Instrumentation Modes
- **Auto-instrumentation** — Zero-code via agents (e.g., `opentelemetry-instrument` CLI)
- **Manual instrumentation** — Explicit spans, metrics, and log records in code

## Collector Architecture

```
App (SDK) → OTLP → [Collector] → Backend (Jaeger / Prometheus / Grafana / Datadog)
                      ├── Receivers
                      ├── Processors (filter, batch, sample)
                      └── Exporters
```

## Python Quick Reference

```python
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider

provider = TracerProvider()
trace.set_tracer_provider(provider)
tracer = trace.get_tracer(__name__)

with tracer.start_as_current_span("llm-inference"):
    result = model.generate(prompt)
```

## LLM / ML Relevance

- Trace LLM API calls (latency, token counts, retries)
- Instrument ML pipelines end-to-end across services
- Monitor feature store lookups, data preprocessing, model inference
- Integrates with **LangSmith**, **Arize**, **Traceloop**, and **MLflow** via OTLP exporters

> [!TIP] OTel + LLM Observability
> Use `opentelemetry-instrumentation-openai` or Traceloop's SDK to auto-capture prompt/completion spans without manual instrumentation.

## Common Backends

| Backend | Signal | Notes |
|---|---|---|
| Prometheus | Metrics | Pull-based scraping |
| Jaeger / Tempo | Traces | Distributed tracing UI |
| Grafana | All | Unified dashboards via OTLP |
| Datadog / Dynatrace | All | Commercial, full-stack APM |
