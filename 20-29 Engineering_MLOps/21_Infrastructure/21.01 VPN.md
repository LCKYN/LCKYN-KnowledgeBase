# VPN (Virtual Private Network)

## Overview
A **VPN** extends a private network across a public network, allowing devices to send/receive data as if directly connected to the private network. Provides **encryption**, **authentication**, and **tunneling** to secure traffic over untrusted infrastructure.

## Key Concepts

- **Tunneling**: Encapsulates packets inside another protocol (e.g., IP-in-IP, GRE, WireGuard).
- **Encryption**: Protects payload confidentiality (AES-256, ChaCha20).
- **Authentication**: Verifies peer identity (PSK, certificates, public keys).
- **Topology**: Client-to-site (remote access) vs. site-to-site (gateway-to-gateway) vs. mesh.

## VPN Types

| Type | Layer | Use Case | Examples |
|------|-------|----------|----------|
| **IPsec** | L3 | Site-to-site, enterprise | strongSwan, Libreswan |
| **WireGuard** | L3 | Modern, fast, simple | WireGuard, Tailscale, Netbird |
| **OpenVPN** | L3/L4 | Cross-platform remote access | OpenVPN, AWS Client VPN |
| **SSL/TLS VPN** | L7 | Browser-based access | Cloudflare Access, Zscaler |
| **SSH Tunnel** | L7 | Ad-hoc port forwarding | `ssh -L`, `ssh -D` |

## Protocol Comparison

| Protocol | Speed | Crypto | Config Complexity | NAT Traversal |
|----------|-------|--------|-------------------|---------------|
| WireGuard | Very High | Modern (Curve25519, ChaCha20) | Low | Good |
| IPsec/IKEv2 | High | AES, SHA2 | High | Moderate |
| OpenVPN | Moderate | TLS-based | Moderate | Good |

## Mesh / Zero-Trust VPNs

Modern overlay-style VPNs build a peer-to-peer mesh on top of WireGuard with identity-based access:
- **Tailscale** - Hosted control plane, magic DNS, ACLs
- **Netbird / Headscale** - Self-hosted alternatives
- **ZeroTier** - Custom L2 overlay protocol

> [!TIP] Practical Pick
> For developer/team access to internal services, prefer **WireGuard-based mesh VPNs** (Tailscale, Netbird) over legacy IPsec gateways—simpler config, NAT-friendly, identity-aware.

## Practical Use Cases

- **Remote access** to private ML training clusters or internal dashboards
- **Site-to-site** linking on-prem GPU servers with cloud control plane
- **Securing data plane** between regions for federated training/inference
- **Bypassing geo-restrictions** for dataset/model downloads (compliance permitting)

> [!WARNING] Common Pitfalls
> - MTU mismatches cause silent throughput collapse—set `MTU=1420` for WireGuard over Ethernet.
> - Split-tunnel misconfig can leak DNS or route private CIDRs to the public internet.
> - Long-lived PSKs without rotation are a credential-leak risk.
