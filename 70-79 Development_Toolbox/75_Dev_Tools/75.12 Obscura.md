# Obscura

## Overview
**Obscura** is an open-source **headless browser** written in **Rust**, built specifically for **AI agents** and **web scraping**. It runs real JavaScript via V8, speaks the **Chrome DevTools Protocol (CDP)**, and works as a drop-in replacement for headless Chrome with **Puppeteer** and **Playwright** — at a fraction of the memory and startup cost.

🔗 **Repo**: [github.com/h4ckf0r0day/obscura](https://github.com/h4ckf0r0day/obscura)

> [!INFO] Why it matters
> Most agent stacks pay the full cost of Chromium (200+ MB RAM, ~2s startup) just to render a page. Obscura targets the automation use case directly — small binary, instant startup, and built-in anti-detection.

---

## Obscura vs Headless Chrome

| Metric | Obscura | Headless Chrome |
|---|---|---|
| Memory | **30 MB** | 200+ MB |
| Binary size | **70 MB** | 300+ MB |
| Startup | **Instant** | ~2 s |
| Page load | **~85 ms** | ~500 ms |
| Anti-detect | **Built-in** | None |
| Puppeteer / Playwright | ✅ via CDP | ✅ |

---

## Key Features
- **Single binary** — no Chrome, no Node.js, no system deps
- **CDP server** (`obscura serve --port 9222`) — connect from Puppeteer / Playwright unchanged
- **Stealth mode** (`--features stealth` / `--stealth`) — fingerprint randomization (GPU, canvas, audio), `navigator.webdriver = undefined`, native function masking, blocks ~3,520 tracker domains
- **Parallel scraping** (`obscura scrape ... --concurrency 25`) — worker-process model
- **DOM-to-Markdown** (`LP.getMarkdown` CDP method) — convenient for LLM ingestion
- **Live request interception** — `Fetch.continueRequest`, `fulfillRequest`, `failRequest`

## Quick Start

```bash
# Fetch + evaluate JS on a page
obscura fetch https://example.com --eval "document.title"

# Dump rendered HTML (waits for network idle)
obscura fetch https://news.ycombinator.com --dump html --wait-until networkidle0

# Start a CDP server with stealth on
obscura serve --port 9222 --stealth

# Parallel scrape with JS evaluation
obscura scrape url1 url2 url3 --concurrency 25 \
  --eval "document.querySelector('h1').textContent" --format json
```

Connect from Playwright as if it were Chrome:

```ts
import { chromium } from "playwright-core";
const browser = await chromium.connectOverCDP({ endpointURL: "ws://127.0.0.1:9222" });
```

## Practical Use Cases
- **LLM agent browsing tools** — give an agent a real browser without paying Chromium's cost; pair with `LP.getMarkdown` for clean context.
- **Large-scale scraping** — 25+ parallel workers per machine thanks to the small footprint.
- **Anti-bot evasion** — stealth mode handles common fingerprint checks out of the box.
- **CI / serverless rendering** — instant startup makes per-request invocation viable.

> [!WARNING] Maturity
> Project is young (v0.1.x at time of writing) and Rust-based — expect rough edges and breaking changes. Treat as an alternative to evaluate, not a Chromium replacement for production-critical flows yet.
