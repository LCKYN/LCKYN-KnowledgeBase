# Edit Distance

## Overview

**Edit Distance** measures the minimum number of operations needed to transform one string into another. It's fundamental to spell checking, DNA sequence alignment, diff tools, and fuzzy matching.

## Key Concepts

### Operation Types

| Operation | Description | Example |
|-----------|-------------|---------|
| **Insert** | Add a character | `cat` → `cart` |
| **Delete** | Remove a character | `cart` → `cat` |
| **Substitute** | Replace a character | `cat` → `bat` |
| **Transpose** | Swap adjacent characters | `ab` → `ba` |

## Distance Metrics Comparison

| Metric | Operations Allowed | Complexity | Use Case |
|--------|-------------------|------------|----------|
| **Hamming** | Substitute only | $O(n)$ | Equal-length strings, error detection |
| **Levenshtein** | Insert, Delete, Substitute | $O(mn)$ | Spell check, typo detection |
| **Damerau-Levenshtein** | + Transpose | $O(mn)$ | Typos (transposition common) |
| **LCS Distance** | Insert, Delete only | $O(mn)$ | Diff tools, version control |
| **Jaro-Winkler** | Character matching + transposition | $O(mn)$ | Name matching, deduplication |

## Levenshtein Distance Algorithm

**Recurrence relation**:

$$
d[i][j] = \min \begin{cases}
d[i-1][j] + 1 & \text{(delete)} \\
d[i][j-1] + 1 & \text{(insert)} \\
d[i-1][j-1] + \mathbb{1}_{s[i] \neq t[j]} & \text{(match/substitute)}
\end{cases}
$$

**Space optimization**: Only need previous row → $O(\min(m,n))$ space

> [!TIP] Implementation Pattern
> ```
> Base cases: d[i][0] = i, d[0][j] = j
> Fill table row by row
> Answer: d[m][n]
> ```

### Complexity

| Aspect | Standard DP | Space-Optimized |
|--------|-------------|-----------------|
| Time | $O(mn)$ | $O(mn)$ |
| Space | $O(mn)$ | $O(\min(m,n))$ |

## Variants and Extensions

### Weighted Edit Distance
Assign different costs to operations:
- Keyboard-aware: `a→s` cheaper than `a→p` (adjacent keys)
- Phonetic: `f→ph` cheaper (sound similar)

### Normalized Edit Distance
$$\text{NED} = \frac{d(s,t)}{\max(|s|, |t|)}$$

Produces value in $[0, 1]$ for similarity comparison.

### Edit Distance with Affine Gap Penalty
Used in bioinformatics: opening gap is expensive, extending is cheap
- $\text{gap cost} = g_o + g_e \cdot (k-1)$ for gap of length $k$

## Practical Use Cases

| Domain | Application |
|--------|-------------|
| **Spell checking** | Suggest corrections within edit distance 2 |
| **Fuzzy search** | Elasticsearch, Lucene fuzzy queries |
| **Bioinformatics** | DNA/protein sequence alignment |
| **Plagiarism detection** | Document similarity |
| **Record linkage** | Deduplication, entity resolution |
| **Git diff** | Finding minimal changes between files |

> [!EXAMPLE] Spell Checker Heuristic
> Most typos are within edit distance 2. Precompute all variations for dictionary words → $O(1)$ lookup.

## Related Algorithms

- **Longest Common Subsequence (LCS)**: Related to edit distance via $d = m + n - 2 \cdot \text{LCS}$
- **Needleman-Wunsch**: Global alignment with custom scoring
- **Smith-Waterman**: Local alignment (find best matching substring)
