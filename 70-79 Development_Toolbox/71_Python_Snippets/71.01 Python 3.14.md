# Python 3.14 New Features

## Overview
Python 3.14 released **October 7, 2025**. Major focus: performance, type system, developer experience.

## Release Highlights

| Category | Features |
|----------|----------|
| **Performance** | Free-threaded (no GIL), subinterpreters, experimental JIT |
| **Type System** | Deferred annotations (PEP 649) |
| **Syntax** | Template strings, simpler exception handling |
| **DX** | Enhanced REPL, better error messages |

## Developer Experience

### Enhanced REPL
- **Syntax highlighting**: Keywords, strings, numbers colored
- **Autocompletion**: `import <TAB>`, `from X import <TAB>`
- **Custom themes**: Configure via `~/.pythonrc.py`

Disable: `NO_COLOR=1` or `PYTHON_BASIC_REPL=1`

### Better Error Messages
- **Typo detection**: `forr` â†’ "Did you mean 'for'?"
- **String issues**: Detect unescaped quotes in strings
- **Runtime context**: More helpful ValueError/TypeError messages

### Live Process Debugging (PEP 768)
Attach debugger to running process:
```bash
sudo python3.14 -m pdb -p <PID>
```

## New Syntax

### Template Strings (PEP 750)
Safe string interpolation with validation layer.

```
t"Hello, {name}"  # Returns Template object, not str
```

**Key Difference from f-strings**:
- F-string: Immediate evaluation to `str`
- T-string: Returns `Template` object for processing

**Use Cases**:
- SQL injection prevention (parameterized queries)
- HTML escaping (XSS prevention)
- Custom DSLs

### Exception Syntax (PEP 758)
Parentheses optional for multiple exceptions:
```python
# Before
except (ValueError, TypeError):

# After (optional)
except ValueError, TypeError:
```

### Finally Warnings (PEP 765)
`return`/`break`/`continue` in `finally` blocks now emit `SyntaxWarning`.

## Type System

### Deferred Annotations (PEP 649)
Annotations evaluated lazily, not at definition time.

**Benefits**:
- Forward references work without quotes
- Circular imports less problematic
- Faster module imports (no eager evaluation)

```python
# Works in 3.14 without quotes!
class Node:
    next: Node  # Self-reference
```

**Access Annotations**:
- `get_annotations(cls, format=Format.VALUE)` - Evaluate
- `get_annotations(cls, format=Format.STRING)` - As strings
- `get_type_hints(func)` - For type hints

## Performance

### Free-Threaded Python (PEP 779)
Official GIL-free build for true multi-threaded parallelism.

| Aspect | With GIL | Without GIL |
|--------|----------|-------------|
| Single-thread | Baseline | 10-15% slower |
| Multi-thread CPU-bound | Single core | All cores |
| Memory | Baseline | ~20% more |

Build: `--disable-gil` or `pyenv install 3.14.0-freethreaded`

### Subinterpreters (PEP 734)
`InterpreterPoolExecutor` - True parallelism without process overhead.

| Executor | Parallelism | Overhead |
|----------|-------------|----------|
| ThreadPool | No (GIL) | Low |
| ProcessPool | Yes | High |
| InterpreterPool | Yes | Medium |

### Experimental JIT
Enable: `PYTHON_JIT=1 python3.14 script.py`

> [!WARNING]
> Experimental, may slow some workloads. For testing only.

### Incremental GC
Garbage collection work spread across multiple steps. Reduces pause times for latency-sensitive apps.

## Standard Library

### Zstandard Compression (PEP 784)
```python
import compression.zstd as zstd
compressed = zstd.compress(data, level=3)
```

### UUID Versions 6-8
- `uuid.uuid6()` - Timestamp-ordered
- `uuid.uuid7()` - Unix Epoch-based
- `uuid.uuid8()` - Custom format
- 40% faster generation for UUIDs 3-5

### Color Support
Colorized output in `unittest`, `argparse`, `json`, `calendar`.

## Migration Notes

| Change | Action |
|--------|--------|
| `from __future__ import annotations` | No longer needed |
| Annotation evaluation | May break code relying on eager eval |
| GIL-free build | Separate ABI, extensions need rebuild |
| `finally` control flow | Add explicit variable handling |

## Deprecations & Removals

- **Removed**: Many deprecated stdlib modules
- **Deprecated**: Some old APIs (see release notes)
- **Changed**: Sigstore replaces PGP for release signing
