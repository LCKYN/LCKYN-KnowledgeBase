# Behavioral Design Patterns

## Overview
**Behavioral patterns** assign responsibilities between objects and define how they communicate. This is the largest GoF family (11 patterns). The recurring move: *turn a behavior into an object (or function) so it can be selected, queued, stored, or swapped at runtime*.

---

## Pattern Summary

| Pattern | Intent | Typical trigger |
|---|---|---|
| **Strategy** | Swap an algorithm behind one interface | `if/elif` choosing between algorithms |
| **Observer** | Notify subscribers when state changes | "When X happens, also do Y and Z" |
| **Command** | Reify a request as an object | Undo/redo, queues, macros |
| **State** | Behavior changes with internal state | State-dependent `if` chains in every method |
| **Template Method** | Skeleton algorithm, subclasses fill steps | Copy-pasted workflows differing in 2 steps |
| **Chain of Responsibility** | Pass a request along handlers until one handles it | Middleware, escalation rules |
| **Iterator** | Sequential access without exposing internals | Built into Python (`__iter__`, generators) |
| **Mediator** | Centralize many-to-many communication | Components talking to each other directly |
| **Memento** | Capture/restore state without breaking encapsulation | Snapshots, undo |
| **Visitor** | New operations over a stable object structure | Many operations on an AST/tree |
| **Interpreter** | Evaluate sentences of a small language | Rule/filter DSLs (rare — prefer a parser lib) |

---

## Strategy

- Encapsulate interchangeable algorithms behind one interface; the context is configured with one and delegates to it.
- In Python a strategy is usually just a **function passed as an argument** — `sorted(data, key=...)` is Strategy in the standard library.

```python
def by_relevance(docs: list[Doc]) -> list[Doc]: ...
def by_recency(docs: list[Doc]) -> list[Doc]: ...

def search(query: str, rank: Callable[[list[Doc]], list[Doc]]) -> list[Doc]:
    return rank(match(query))
```

- [[71.10 Policy Pattern over Boolean Flags]] is Strategy generalized to a configurable *list* of behaviors.

---

## Observer

- A **subject** keeps a list of subscribers and notifies them on events; publishers don't know who listens.
- Foundation of event systems, reactive UIs, and pub/sub messaging (see [[31.02 Serving Architecture]] for the distributed version).

```python
class Signal:
    def __init__(self): self._subs: list[Callable] = []
    def connect(self, fn): self._subs.append(fn)
    def emit(self, *a): [fn(*a) for fn in self._subs]
```

> [!WARNING] Hidden control flow
> Heavy observer use makes "what happens when I save?" hard to answer — execution order and side effects hide in subscriber lists. Keep handlers few and idempotent.

---

## Command

- Wrap a request (receiver + action + args) in an object so it can be **queued, logged, undone, or replayed**.
- Pythonic form: a `Callable` or `functools.partial`; add an `undo()` method when you need reversibility.
- Enables: undo stacks (store executed commands), task queues (serialize commands), macro recording.

---

## State

- An object delegates behavior to a **state object** and swaps it on transitions — each state class handles events its own way.
- Replaces `if self.status == "paid": ... elif self.status == "shipped": ...` repeated across methods.
- For heavy workflows, prefer an explicit state-machine library (`transitions`) over hand-rolled State classes.

---

## Template Method

- A base class defines the **skeleton** (`load → validate → transform → save`); subclasses override individual steps.
- Inheritance-based; the composition-based alternative is Strategy (inject the varying step as a function). Prefer Strategy when the varying part is small.

---

## Chain of Responsibility

- Handlers linked in a chain; each either handles the request or passes it on.
- Real-world forms: WSGI/ASGI **middleware**, logging handler chains, support-ticket escalation.

---

## Iterator, Mediator, Memento, Visitor, Interpreter (brief)

- **Iterator** — native in Python: implement `__iter__`/`__next__` or just write a **generator**. Never hand-roll GoF iterator classes.
- **Mediator** — components talk to a hub, not each other; cuts N² coupling in dialogs/orchestrators. The orchestrator in a pipeline is a mediator.
- **Memento** — snapshot state for undo/rollback without exposing internals; Python: `dataclasses.asdict` or `pickle` of an immutable state object.
- **Visitor** — add operations to a stable node hierarchy without editing node classes; Python's `ast.NodeVisitor` is the canonical example. Awkward without double dispatch — often a `match` statement is clearer.
- **Interpreter** — class-per-grammar-rule evaluation of a mini-language; in practice use `ast`, a parsing library, or a rules engine instead.

---

## Choosing Between Look-alikes

| Confusion | Rule of thumb |
|---|---|
| Strategy vs State | Caller picks a Strategy; the object switches its own State |
| Strategy vs Template Method | Composition (inject function) vs inheritance (override step) |
| Observer vs Mediator | Broadcast to unknown listeners vs coordinated hub |
| Command vs Strategy | Command = *a* request reified; Strategy = *how* to do one task |
