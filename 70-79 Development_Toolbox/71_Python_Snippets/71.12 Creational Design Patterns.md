# Creational Design Patterns

## Overview
**Creational patterns** decouple *what* an object is from *how and when it gets created*. They matter once construction becomes complex: many configuration steps, families of related objects, expensive setup, or the concrete class only known at runtime.

---

## Pattern Summary

| Pattern | Intent | Reach for it when… |
|---|---|---|
| **Singleton** | Exactly one instance, globally reachable | Shared config, connection pool, logger |
| **Factory Method** | Subclass/parameter decides which class to instantiate | Concrete type known only at runtime |
| **Abstract Factory** | Create whole *families* of related objects | Swappable backends (Postgres vs SQLite, light vs dark UI) |
| **Builder** | Assemble a complex object step by step | Many optional parts; construction order matters |
| **Prototype** | Clone an existing object instead of constructing | Construction is expensive; copies with small tweaks |

---

## Singleton

- Guarantees **one instance** and a global access point.
- In Python, prefer a **module** — modules are singletons by design — or `functools.lru_cache` on a factory function.

```python
@lru_cache(maxsize=1)
def get_settings() -> Settings:
    return Settings()  # built once, shared everywhere
```

> [!WARNING] Singleton is a global in disguise
> It hides dependencies and couples tests to shared state. Prefer passing the instance explicitly (see Dependency Injection in [[71.15 Advanced Design Patterns]]); use a true singleton only for genuinely process-wide resources.

---

## Factory Method

- A creator exposes one method that returns a **product interface**; the concrete class is chosen by a subclass or a parameter.
- Pythonic form: a plain function (or `classmethod`) with a **registry dict** — no class hierarchy needed.

```python
LOADERS: dict[str, type[Loader]] = {"csv": CsvLoader, "parquet": ParquetLoader}

def make_loader(fmt: str) -> Loader:
    return LOADERS[fmt]()
```

> [!TIP] Registry + entry points
> A dict registry keeps creation **open for extension** (register a new format) but **closed for modification** — the OCP applied to construction. Same idea as the registry in [[71.10 Policy Pattern over Boolean Flags]].

---

## Abstract Factory

- One factory object produces a **consistent family** of products (`make_connection()`, `make_cursor()`, `make_migrator()`), so mixed families are impossible.
- Client code depends only on the factory interface — swap the whole backend by swapping one object.

> [!EXAMPLE]
> A `StorageFactory` protocol with `S3Factory` and `LocalFactory` implementations; the app receives one factory at startup and never mentions S3 or the filesystem again.

- **vs Factory Method**: Factory Method creates *one* product; Abstract Factory creates *several that must match*.

---

## Builder

- Separates construction of a complex object from its representation; the same steps can build different results.
- Useful when a constructor would need many optional arguments or when parts must be assembled in order (SQL queries, HTML documents, request objects).

```python
query = (QueryBuilder("events")
         .where("ts > :start")
         .order_by("ts")
         .limit(100)
         .build())
```

- Pythonic alternatives: keyword-only arguments with defaults, or a `dataclass` + `dataclasses.replace` for stepwise refinement. Reach for Builder only when construction has *logic*, not just many fields.

---

## Prototype

- Create new objects by **copying a pre-configured instance** rather than building from scratch.
- Python: `copy.deepcopy`, or `dataclasses.replace(base, **overrides)` for a cheap "clone with tweaks".

```python
base_run = RunConfig(lr=3e-4, epochs=10, seed=42)
sweep = [replace(base_run, lr=lr) for lr in (1e-4, 3e-4, 1e-3)]
```

---

## Choosing Between Them

- Concrete class varies → **Factory Method**.
- Families of objects must stay consistent → **Abstract Factory**.
- Construction is multi-step with options → **Builder**.
- Instances are near-copies of a template → **Prototype**.
- Exactly one shared instance → **Singleton** (but prefer a module or DI).
