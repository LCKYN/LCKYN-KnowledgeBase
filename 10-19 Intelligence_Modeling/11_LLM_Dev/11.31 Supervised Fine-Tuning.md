# 11.31 Supervised Fine-Tuning

## Overview
**Supervised Fine-Tuning (SFT)** adapts a pre-trained base LLM to follow instructions or specialize on a domain by training on curated **(prompt, response)** pairs using standard cross-entropy loss. It is the first post-training step before alignment (RLHF/DPO).

## Key Concepts
- **Objective**: Next-token prediction (causal LM loss), but **only on response tokens** — prompt tokens are masked out
- **Data format**: Chat templates (e.g., ChatML, Llama-3, Qwen) wrap turns with role tokens
- **Loss masking**: `labels = -100` on system/user/special tokens so gradient flows only through assistant output
- **Packing**: Concatenate multiple short examples into one sequence to maximize GPU utilization
- **Epochs**: Typically **1–3 epochs** — more risks catastrophic forgetting and overfitting

## SFT vs Related Stages

| Stage | Data | Signal | Purpose |
| --- | --- | --- | --- |
| Pre-training | Raw web text | Next-token loss on all tokens | World knowledge |
| **SFT** | Curated (prompt, response) | Next-token loss on response only | Instruction following, format |
| RLHF / DPO | Preference pairs (chosen, rejected) | Reward / preference loss | Alignment, helpfulness, safety |

## Data Quality Principles
> [!TIP] Quality > Quantity
> LIMA showed **1,000 high-quality examples** can rival 50k mediocre ones. Prioritize:
> - Diverse instruction types (reasoning, extraction, creative, refusal)
> - Consistent format and tone
> - Correct, complete responses
> - Deduplication across prompts

- **Synthetic data**: Self-Instruct, Evol-Instruct, distillation from stronger models
- **Human-curated**: Dolly, OpenAssistant, ShareGPT
- **Domain-specific**: Code (CodeAlpaca), medical, legal

## Practical Use Cases
- Turning a base model (e.g., Llama-3 base) into a chat assistant
- Teaching a fixed output format (JSON, function calls, XML tags)
- Domain adaptation (medical Q&A, code in a niche language)
- Style/persona transfer
- Bootstrapping reward models for [[11.32 RLHF]]

## Common Recipes
- **Full fine-tuning**: Update all weights — best quality, highest VRAM
- **LoRA / QLoRA**: Train low-rank adapters — 10–100× cheaper, near-parity for most tasks
- **Hyperparameters**: LR ~1e-5 to 2e-5 (full FT), ~1e-4 to 3e-4 (LoRA); cosine schedule; warmup ~3%
- **Batch size**: Effective batch 64–256 sequences via gradient accumulation

> [!WARNING] Common Pitfalls
> - **Forgetting**: Too many epochs or too-high LR erodes pre-trained knowledge
> - **Loss on prompts**: Forgetting to mask user tokens leaks gradient and hurts quality
> - **Template mismatch**: Training and inference must use **identical** chat templates
> - **Distribution shift**: SFT data far from pre-training distribution causes brittleness
