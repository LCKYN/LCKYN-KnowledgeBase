# GRPO

## Overview
**Group Relative Policy Optimization (GRPO)** is a reinforcement-learning algorithm for LLM post-training that removes PPO's value (critic) model. For each prompt it samples a **group** of responses, scores them, and uses the group's mean score as the baseline — each response's advantage is simply how much better or worse it did *relative to its siblings*. Introduced by DeepSeek (DeepSeekMath, then DeepSeek-R1), it made large-scale reasoning RL dramatically cheaper.

## How It Works

1. For each prompt, sample **G** responses (e.g., G = 8–64) from the current policy
2. Score each response with a reward (learned [[11.50 Reward Models|RM]] or a verifiable checker)
3. Compute **group-relative advantage**: $A_i = \frac{r_i - \text{mean}(r_{1..G})}{\text{std}(r_{1..G})}$
4. Update the policy with a PPO-style clipped objective + KL penalty — but no critic to estimate baselines

> [!INFO] Why the group baseline works
> PPO needs a critic network to estimate "how good is this state on average" for variance reduction. GRPO gets the same effect empirically: siblings sampled from the same prompt *are* the baseline. One fewer full-size model to train and hold in memory.

## GRPO vs PPO

| | **GRPO** | **PPO** |
|---|---|---|
| Value/critic model | None | Full policy-sized network |
| Baseline | Group mean of sampled rewards | Learned per-token value estimate |
| Memory | ~25–40% less | 4 models resident |
| Samples per prompt | Many (a group) | Typically 1 |
| Best fit | Verifiable rewards, reasoning tasks | Dense/learned rewards, general RLHF |

## RLVR: The Killer Combination

GRPO shines with **Reinforcement Learning from Verifiable Rewards (RLVR)** — rewards computed by a program, not a model:

- **Math** — answer matches ground truth (or symbolic check)
- **Code** — unit tests pass
- **Format** — output parses / follows required structure

No reward model → no reward hacking of a learned proxy ([[11.50 Reward Models]]), and group sampling naturally explores multiple reasoning paths. This is the recipe behind **DeepSeek-R1**'s emergent long [[11.13 Chain of Thought]] reasoning.

> [!WARNING] Pitfalls
> - **Zero-variance groups** — if all G responses get the same reward (all right or all wrong), advantage is 0 and the prompt teaches nothing; curriculum/difficulty filtering matters
> - **Reward sparsity** — binary pass/fail on hard tasks gives rare signal; needs many samples
> - **Length & entropy drift** — unconstrained reasoning RL can balloon response length; most recipes add length or entropy controls (DAPO and successors tune these details)

## Practical Use Cases

- Training **reasoning models** on math/code with checkable answers
- Post-training where you can write a verifier but can't afford preference labeling
- Available in TRL, veRL, OpenRLHF; pairs with [[11.37 Distributed Training]] since group sampling is generation-heavy
