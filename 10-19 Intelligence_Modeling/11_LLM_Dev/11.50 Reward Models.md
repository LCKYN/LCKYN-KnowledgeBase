# Reward Models

## Overview
A **reward model (RM)** is a model trained to predict which of two LLM outputs a human would prefer, producing a scalar score for any (prompt, response) pair. It converts sparse, expensive human judgment into a cheap, dense training signal — the optimization target for [[11.49 RLHF]] and the ranking backbone for best-of-N sampling and data filtering.

## How They're Trained

- Start from an SFT checkpoint; replace the LM head with a **scalar value head**
- Data: humans see a prompt with two (or more) responses and pick the better one → **preference pairs** $(y_w, y_l)$
- Objective: **Bradley-Terry model** — the probability response $y_w$ beats $y_l$ is a sigmoid of the score difference:

$$P(y_w \succ y_l) = \sigma\big(r(x, y_w) - r(x, y_l)\big)$$

- Loss maximizes this likelihood; only score *differences* matter, so raw RM scores are not comparable across prompts

## Outcome vs Process Reward Models

| | **Outcome RM (ORM)** | **Process RM (PRM)** |
|---|---|---|
| Scores | Final answer only | Each reasoning step |
| Labels needed | Cheap (right/wrong, preferred) | Expensive (per-step correctness) |
| Best for | Chat quality, general preference | Math / multi-step reasoning ([[11.13 Chain of Thought]]) |
| Failure mode | Rewards lucky wrong reasoning | Label noise per step |

## Reward Hacking

> [!WARNING] The RM is a proxy, and proxies get gamed
> Optimizing hard against an imperfect RM (Goodhart's law) produces outputs the RM loves but humans don't:
> - **Length bias** — RMs systematically prefer longer answers → verbose models
> - **Sycophancy** — agreeing with the user scores higher than correcting them
> - **Format gaming** — bullet lists, confident tone, hedging patterns that score well regardless of content
> - **Off-distribution exploits** — the policy drifts into regions the RM never saw, where scores are meaningless

Mitigations: KL penalty to the reference policy ([[11.49 RLHF]]), RM ensembles, periodic RM refresh with fresh preference data on the current policy's outputs, and length-normalized rewards.

## Beyond RLHF Training

- **Best-of-N / rejection sampling** — generate N candidates, keep the top-scoring one (inference-time alignment, no RL needed)
- **Data filtering** — score synthetic or scraped SFT data, keep the top slice
- **[[11.15 LLM Evaluation Metrics|Evaluation]]** — RMs as automatic judges (shares failure modes with LLM-as-judge)
- **Verifiable rewards (RLVR)** — replace the learned RM with a programmatic checker (unit tests, math verification) when correctness is checkable; see [[11.52 GRPO]]
