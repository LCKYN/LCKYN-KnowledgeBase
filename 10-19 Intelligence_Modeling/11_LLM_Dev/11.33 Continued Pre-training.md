# 11.33 Continued Pre-training

## Overview
**Continued Pre-training (CPT)** — also called **Domain-Adaptive Pre-Training (DAPT)** — extends a base model's pre-training on a large, **unlabeled** domain corpus using the original next-token objective. It injects new knowledge or vocabulary before any [[11.31 Supervised Fine-Tuning|SFT]] or alignment.

## Key Concepts
- **Objective**: Same causal LM loss as pre-training (loss on **all tokens**, unlike SFT)
- **Data**: Raw text — code, papers, legal docs, non-English corpora — usually **billions of tokens**
- **No labels needed**: Unsupervised, so easier to scale than SFT
- **Position in pipeline**: Base → **CPT** → SFT → DPO/RLHF
- **Scale**: Typically 1–10% of original pre-training tokens (e.g., 50B–500B for a Llama-class model)

## CPT vs SFT vs Pre-training

| Stage | Data | Loss On | Goal |
| --- | --- | --- | --- |
| Pre-training | Raw web text (trillions of tokens) | All tokens | Broad world knowledge |
| **CPT** | Raw domain text (billions of tokens) | All tokens | Inject domain knowledge / language |
| SFT | (Prompt, response) pairs (thousands–millions) | Response tokens only | Instruction following, format |

## When to Use CPT
- **Low-resource languages**: Adapting an English-centric base to Thai, Vietnamese, etc.
- **Specialized domains**: Medical (PubMed), legal (case law), code (private repos), finance
- **New vocabulary**: Adding domain-specific tokens via tokenizer extension
- **Knowledge cutoff updates**: Refreshing facts (though RAG is usually cheaper — see [[11.12 RAG]])

> [!TIP] CPT vs RAG vs SFT — Choosing
> - **RAG**: Facts that change, need citations, narrow lookup
> - **SFT**: Format, style, task behavior
> - **CPT**: Deep domain fluency, new language, large-scale knowledge that won't fit in context

## Practical Recipe
- **Learning rate**: Much lower than original pre-training (~1e-5 to 5e-5); use **warmup + cosine decay**
- **Data mixing**: Blend domain data with **5–30% replay** of original pre-training data to mitigate catastrophic forgetting
- **Tokenizer**: Extend vocabulary for new languages/scripts; resize embeddings and **initialize new tokens from mean of subword pieces**
- **Sequence packing**: Same as pre-training — pack to full context length
- **Duration**: 1 epoch is often enough on large corpora; multiple epochs risk overfitting

## Catastrophic Forgetting
> [!WARNING] The Core Risk
> CPT can **degrade general capabilities** (reasoning, English fluency, instruction following) if domain data dominates. Mitigations:
> - **Replay buffer**: Mix in original pre-training data
> - **LoRA-based CPT**: Train low-rank adapters instead of full weights
> - **Lower LR**: Reduces drift from base distribution
> - **Eval gates**: Run general benchmarks (MMLU, HellaSwag) alongside domain evals every N steps

## Examples in the Wild
- **Code Llama**: Llama-2 → +500B code tokens → instruction tuning
- **MediTron**: Llama-2 → PubMed + clinical guidelines
- **Typhoon, SeaLLM**: English base → Southeast Asian language CPT
- **DeepSeek-Coder**: Trained from scratch but later variants use CPT for math/code refresh

> [!INFO] CPT Often Precedes SFT
> A common production pipeline: **Open base → CPT on domain → SFT on domain instructions → DPO on preferences**. Skipping CPT when domain knowledge is sparse in the base model leads to weak SFT results.
