# OWASP LLM Top 10

## Overview
The **OWASP Top 10 for LLM Applications** is the industry-standard taxonomy of security risks specific to LLM-based systems, maintained by the OWASP GenAI Security Project. It's the shared vocabulary for threat-modeling an LLM app — useful as a checklist when reviewing any system that combines models, retrieval, and tools.

## The Ten Risks (2025 edition)

| # | Risk | One-liner | Vault coverage |
|---|---|---|---|
| LLM01 | **Prompt Injection** | Untrusted text overrides app instructions | [[11.53 Prompt Injection]] |
| LLM02 | **Sensitive Information Disclosure** | Model leaks PII, secrets, or proprietary data in outputs | — |
| LLM03 | **Supply Chain** | Poisoned/backdoored models, datasets, adapters, plugins | — |
| LLM04 | **Data and Model Poisoning** | Tampering with training or retrieval data | [[11.47 Knowledge Base Poisoning]] |
| LLM05 | **Improper Output Handling** | Model output executed/rendered without validation (XSS, SQLi, RCE) | — |
| LLM06 | **Excessive Agency** | Agent has more tools/permissions/autonomy than needed | [[11.54 Agent Tool Sandboxing]] |
| LLM07 | **System Prompt Leakage** | Secrets or logic hidden in system prompts get extracted | — |
| LLM08 | **Vector and Embedding Weaknesses** | RAG-specific: poisoned or cross-tenant-leaking vector stores | [[11.47 Knowledge Base Poisoning]] |
| LLM09 | **Misinformation** | Confident hallucinations users act on | [[11.46 Falsification-Verification RAG (FVA-RAG)]] (mitigation) |
| LLM10 | **Unbounded Consumption** | Denial-of-wallet: runaway tokens, cost, or model extraction | — |

## Key Themes Across the List

- **The trust-boundary triad** — LLM01, LLM04, LLM08 are all "untrusted text reaches the model"; root cause in [[11.48 User Message Confusion]]
- **Output is untrusted too** — LLM05: treat model output like user input; never `eval`, render, or execute it unvalidated
- **Capability, not content** — LLM06 is an architecture problem; solved by design ([[11.54 Agent Tool Sandboxing]]), not by prompts
- **Never put secrets in prompts** — LLM07: assume any system prompt can be extracted; keep authorization server-side
- **Cost is a security surface** — LLM10: rate limits and budgets per user/session, or attackers spend your money

> [!TIP] Using it in practice
> Threat-model per data flow, not per model: for each place text enters (users, RAG, tools) and leaves (UI, tool calls, logs), walk the ten and ask which apply. Most real apps concentrate risk in LLM01, LLM05, and LLM06.

> [!WARNING] What the list is not
> It covers *application* security, not model alignment (jailbreak robustness, harmful content) and not general infra security — your normal OWASP Top 10, authn/authz, and secrets management still apply underneath.
