# 11.38 Flash Attention

## Overview
**Flash Attention** is an **IO-aware exact attention algorithm** that computes standard softmax attention with **linear (not quadratic) memory** and 2–4× speedup by **tiling** the computation to keep working sets in fast on-chip SRAM and avoid materializing the $N \times N$ attention matrix in HBM.

## The Problem It Solves
Standard attention computes $\text{softmax}(QK^T / \sqrt{d}) V$:
- Materializes $S = QK^T \in \mathbb{R}^{N \times N}$ in HBM
- Memory: $O(N^2)$ — prohibitive for long sequences
- Bottleneck: **HBM bandwidth**, not FLOPs (attention is **memory-bound** for typical $d$)

## Key Insight
> [!INFO] Tile, Recompute, Stream
> GPUs have ~100× more SRAM bandwidth than HBM bandwidth. By **tiling** Q, K, V into blocks that fit in SRAM and computing attention block-by-block with **online softmax**, Flash Attention avoids reading/writing the $N \times N$ matrix to HBM at all.

- **Forward**: Stream Q blocks; for each, iterate K/V blocks; maintain running softmax statistics
- **Backward**: Recompute attention from saved $O$, $L$ (logsumexp) — cheaper than storing $S$
- **Memory**: $O(N)$ instead of $O(N^2)$
- **Exact**: Bit-equivalent to standard attention, not an approximation

## Versions

| Version | Year | Key Improvements | Hardware |
| --- | --- | --- | --- |
| FlashAttention-1 | 2022 | Tiling, online softmax, no $N^2$ HBM | A100 |
| FlashAttention-2 | 2023 | Better work partitioning, ~2× faster | A100, H100 |
| FlashAttention-3 | 2024 | Async + FP8 (Hopper TMA, WGMMA) | H100 |
| FlashAttention-4 | 2025 | Blackwell tuning, more head dims | B200 |

## Performance
| Metric | Standard Attention | Flash Attention |
| --- | --- | --- |
| Memory complexity | $O(N^2)$ | $O(N)$ |
| Speedup (8k seq, A100) | 1× | ~3× |
| Long context (32k+) | OOM | Works |
| Forward + backward | Both materialize $S$ | Neither materializes $S$ |

## Variants & Extensions
- **FlashAttention with masking**: Causal, sliding window, custom masks
- **FlashDecoding**: Optimized for **inference** with KV cache (long context, single-query)
- **FlashDecoding++**: Better parallelism for very long sequences at inference
- **Paged Attention** (vLLM): Combines Flash Attention with paged KV cache management
- **Ring Attention**: Sequence-parallel Flash Attention across GPUs for million-token contexts

## Practical Use Cases
- **Default attention kernel** in modern training (PyTorch SDPA dispatches to it automatically)
- **Long-context training**: 32k–1M token sequences become feasible
- **Inference acceleration**: FlashDecoding speeds up [[11.27 LLM Generation Engines]] like vLLM, SGLang
- **Memory-constrained fine-tuning**: Frees activation memory for larger batches

## Integration
- **PyTorch**: `torch.nn.functional.scaled_dot_product_attention` (auto-selects backend)
- **HuggingFace Transformers**: `attn_implementation="flash_attention_2"`
- **xFormers**: Memory-efficient attention with similar approach
- **Triton**: Custom Flash Attention kernels for new variants

> [!TIP] Just Use SDPA
> In modern PyTorch (2.x+), `F.scaled_dot_product_attention` auto-dispatches to Flash Attention when available. You usually don't need to install or call it explicitly.

> [!WARNING] Limitations
> - **Head dim constraints**: Originally limited to powers of 2 up to 128; FA-3+ supports up to 256
> - **Custom attention patterns**: Arbitrary sparse masks may not benefit; use specialized kernels
> - **Older GPUs**: Pre-Ampere (e.g., V100, T4) get limited or no support
> - **Numerical differences vs naive**: Bit-different from a naive PyTorch implementation due to reduction order — usually irrelevant but can confuse unit tests
